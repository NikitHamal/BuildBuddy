# BuildBuddy Android

BuildBuddy is a native Android app (`com.build.buddyai`) for on-device Android project creation, coding, AI-assisted editing, and BuildBuddy-compatible build/install workflows.

## Stack

- Kotlin + Jetpack Compose + Material 3 (Neo Vedic adaptation)
- Hilt dependency injection
- Room local persistence
- DataStore preferences
- WorkManager build orchestration
- PackageInstaller session-based APK install
- Multi-provider AI integration (OpenRouter, NVIDIA, Gemini)

## Core Product Areas

- Splash + one-time onboarding
- Home dashboard (search/sort, create/import, project cards)
- Project creation wizard with validation and templates
- Project playground:
  - Overview
  - Agent (chat/plan/apply/auto-apply)
  - Editor (real EditText-based code editor + highlighting)
  - Files
  - Build
  - Artifacts
- Settings and provider management

## Build Scope Policy

BuildBuddy does not fake full Gradle parity on-device. The build engine:

- validates BuildBuddy project compatibility
- checks toolchain/runtime availability
- streams progress and logs
- returns explicit unsupported diagnostics when runtime/toolchain is missing
- only reports success when a real APK artifact is generated by compiler runtime

## Security and Privacy

- API keys are stored with `EncryptedSharedPreferences`
- no keystore committed to repository
- release signing in CI is secret-driven
- backup rules exclude secure key storage
- install flow requires explicit user confirmation via platform PackageInstaller

## Local Setup

1. Open in Android Studio (JDK 17).
2. Ensure Android SDK 35 platform and matching build-tools are installed.
3. Sync Gradle project.
4. Run `app` on device/emulator.
5. Configure AI providers in `Settings -> Model Providers`.

## CI Release APK

Workflow: `.github/workflows/release-apk.yml`

- Trigger: every `push` on all branches
- No `pull_request` trigger
- Builds signed `release` APK
- Renames artifact to: `BuildBuddy-<shortsha>-release.apk`

Required repository secrets:

- `BUILD_BUDDY_KEYSTORE_BASE64`
- `BUILD_BUDDY_KEYSTORE_PASSWORD`
- `BUILD_BUDDY_KEY_ALIAS`
- `BUILD_BUDDY_KEY_PASSWORD`

## Notes

- This repository intentionally avoids committing production keystore material.
- On-device build success depends on compatible runtime/toolchain availability.
